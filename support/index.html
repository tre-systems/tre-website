Why this exists

A portfolio is useful. A living workshop is better.

TRE projects are deliberately real: they run in browsers, talk to production services, stress GPUs, test agent workflows, teach language skills, create music, simulate physics, and explore game systems. That makes them better than static case studies, but it also means they need care.

Support buys continuity. It lets me keep applications free to try, improve the strongest ones, retire rough edges, and publish more of the engineering process instead of hiding everything behind private client work.
